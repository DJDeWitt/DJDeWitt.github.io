Time for Class

An introduction to class methods and the power of object-oriented programming

October 26, 2014

As we’ve discussed previously, we write software to make our life simpler, doing things that it would be inefficient or impossible for us to do manually. Doing arithmetic equations by hand would get tedious, so we create a Calculator class with individual methods for addition, subtraction, etc. This is how classes and methods, a fundamental feature of object-oriented programming languages, helps us simplify our lives using code.

These building blocks – classes, methods, variables – come in a variety of shapes and sizes, giving programmers a greater palette of solutions to their problems. For example, different types of variables can have different scopes; class variables can be accessed by every object of a class, whereas instance variables can only be accessed by that particularly object. This diversity gives programmers greater control over the code they write, which, in turn, allows them to write more versatile and efficient code.

Like variables, methods also can come in a variety of forms. The type of method you’re probably most familiar with is the instance method, which looks like this: 

class Calculator
  def add_numbers(a,b)
    c = a + b
  end
end

As you can see, an instance method is simply defined using the regular “def” syntax. To access an instance method, you simply have to create a new object of the class and then call the method using that object. 

my_calc = Calculator.new # creates a new object of the Calculator class and assigns it to my_calc
my_calc.add_numbers(3,4) # returns 7 (the sum of 3 and 4)

Much in the way that there is a class variable and an instance variable, we also have a class method to go along with our instance method. This is how you create, and then call, a class method. 

class Calculator
  @@total_calculators = 0

  def initialize
    @@total_calculators +=1
  end

  def add_numbers(a,b)
    c = a + b
  end

  def self.calculator_count
    @@total_calculators
  end
end

Calculator.calculator_count # will return the value of the class variable total_calculators

As you can see, we’ve added a new method, but this time we affixed self to the beginning of our method name. This is how we turn an instance method into a class method. This method, in particular, returns a class variable that will contain the total number of Calculator objects we’ve created. To do so, we had to make sure we increment this class variable by 1, every time a new object is initialized.

You will more frequently use instance variables over class variables, and the same can be said about instance methods and class methods. That said, class methods can help store, manipulate, and access data about an entire class (class methods are accessible by instance methods), giving you significant control over an entire class that can allow you to quickly and efficiently maintain your instances. It may not seem that valuable when you’re only working with singular instances of a class, but when your programs get more complex, you’ll be glad you have class methods as a resource.
