Find your way around the Ruby .map method

October 14, 2014

As we learned in our last lesson, most software involves manipulation of data, and arrays and hashes are powerful tools for storing, manipulating, and retrieving this data. In this lesson, we’ll talk more specifically about the last two of those verbs: data manipulation and data retrieval. We’ll do this through a wonderful little enumerable method called #map.

Say we have an array of test scores for a class we’re teaching:

class_scores = [80,65,79,92,66,84,89]

Now let’s say we decide we’re going to give everyone in the class five points of extra credit (we’re super cool teachers). We could manually add five points to every score in our list, but that would be inefficient. Instead, we’ll use the #map method.

class_scores.map! { |score| score += 5 }

Okay, let’s go over what we’re doing here. First, we’ve got the name of our array, then we’re calling the map method on it using dot notation (note, we add the “!” mark because we want to modify the original array).

Now here’s where the real power of #map gets in. You’ll notice the open and close curly braces after our method, with some stuff inside those curly braces. This is what we like to call a block. Basically the #map method runs whatever is inside this block on every element in the array. In this block, specifically, we’re first defining a variable to store the element (|score|) then adding five to this value using score += 5.

Of course, running whatever is inside the block on a collection of values (stored in an array or hash) gives us near unlimited potential for efficient data manipulation. Heck, we can even have a print function inside the block, and use the map function for quick data retrieval. Ultimately, #map gives you greater control over your data than you’ve ever had before.
